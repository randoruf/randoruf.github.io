Container

Container is a set holds a collection of objects of the same type.

string is not a container type. But it supoorts most of container operations.

但是注意, 虽然 Container 有很多 Member method, 但是依然会有 Buffer Overflow 的问题， 并不会自带检查功能。

Why Iteratror? - More General

From Python and Matlab shit, you will be more farmilar with subscript and range-based for

Example 1: subscript

indexing to access elements in container. 

vector<int> ivec{0, 1, 2, 3}; 
for (decltype(ivec.size()) i = 0; i != ivec.size(); ++i){
    ivec[i] *= 2;  
}

Example 2: range-based for loop

Very useful in for loop 

vector<int> ivec{0, 1, 2, 3}; 
for (auto &i : ivec){
    // reference binding allowws modification 
    i *= 2; 
}

But here is a sad news. No all containter types support subscript and range-based for.

But C++ has the convention that

All library containter types have iterators.

Example 3: Iterator does the same job !

Yes, iterator does the same job as what subscript and range-based for will do. You should always use iterator. 

vector<int> ivec{0, 1, 2, 3}; 
for (auto it = ivec.begin(); it != ivec.end(); ++it){
    // `it` is an iterator 
    // *it will return the reference to the element, so you can modify it by assignment. 
    *it *= 2; 
}

Note that end() will return an iterator positions one past the end which is a nonexistent element of the containter. See it is very convienent to write it.

Vector and For Loop

If the size of a vector will change in a for loop (like adding or deleting elements inside the for loop), some unexpected behaviours may occur.

Thus, be careful if the size of a vector will change. You could use the while loop instead. 

size_type i = 0; 
while (i != vec.size()){
    // some operation
    i++; 
}
