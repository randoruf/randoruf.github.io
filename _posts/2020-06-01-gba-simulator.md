记得以前看王明达的帖子， 说苏联的教育喜欢往学生的脑子的塞东西， 所以教科书内容特别多， 题目特别多，通常学完一个科目需要好几年的时间。 而美国人更注重从实践中学习， 先立项，如果遇到问题了再学习。 最后证明，美国人的学习方法可以让人更专注。 

其实也可以这种， 你可以以做一个游戏、一台GBA游戏机、一辆类似ECE3091的自动寻路车等等各种小项目为目标。 

例如一个游戏可以学习 OpenGL ，计算机图形， 3D模型设计等知识。 

一台GBA游戏机可以学到PCB板设计， 树莓派， JVM虚拟机， 操作系统， Solidwork 等等的复杂繁琐的知识。

一辆ECE3091可以用到 Computer Vison SIFT, CNN, 还有如何顺滑移动的 feedback control（应该还有更简单的方法， 我大二的时候做的车子像残疾人一样爬一爬停一停， 连詹天扬都说耻辱） 

还有詹天扬做过的自平衡车， 自动贩卖机， （当然可以用树莓派做， 要考虑好如何检测硬币， 纸币，与刷卡机连接）。 

更别说用 JavaScript 做各种小游戏， 教学网页， 打飞机， 日本人喜欢玩的弹珠游戏等等。 还有 JavaScript 是有3D的， 你还可以把 Windows 上的小游戏全部在 JavaScript 上面实现一次， 还有手机上的小游戏也是， 甚至还有触屏时代刚开始时的切水果， 愤怒的小鸟。



> 只要你想，计算机可以模拟一切， 这才是计算机的魅力。 

这也是计算机的正真的实力。 唯一阻碍你成长的， 是你惰性。

你级别越来越高以后，就去做一些开源项目的维护者，这个世界太多东西等着你去做了。
