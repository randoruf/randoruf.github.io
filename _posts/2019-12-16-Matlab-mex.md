## 1. GCC: Compiling C/C++ programs 

**C/C++ is not cross-platform**. You would better to provide the source code to take care of other computer architecture (for example, x86 and AMD64 even the latest ARM CPU used by Microsoft Surface Pro X). 

 ![](/shared/imgs/Makefiles.gif)

Compilation of C <http://courses.cms.caltech.edu/cs11/material/c/mike/misc/compiling_c.html>

* Pre-processor
  * start with `#` such as `#include` and `#define`
  * `#include` to access function definitions defined outside of the source code file
  * `#define` to define some constants 
* Compiler 
  * **Red Hat Enterprise Linux 4: Using the GNU Compiler Collection (GCC)**
  * <https://access.redhat.com/documentation/en-US/Red_Hat_Enterprise_Linux/4/html/Using_the_GNU_Compiler_Collection/invoking-gcc.html>
  * `gcc -c -o` 
    * `-c` tells the compiler to compile without any linking. 
    * `-o ` specify the output location and output name, if you don't specify the `-o`, the object file will be named as `a.out`
* Linker 
  * Linker will link together a bunch of object files (`.o` files) into a binary executable. 
    * Include the object files generated by your functions, and from **library files** (such as `-lm` will link the math library).
  * `gcc -Wall x.o y.o z.o -o xyz`
    * reporting all warnings `-Wall`
    * more restrict (even more anal, but probably unnecessary) `-Wstrict-prototypes` , `-ansi`, `-pedantic`

---

* **Object** files
  * Object code is not directly executable. (having dependencies on other header files or functions)  
  * usually end in with `.o` by convention, but in Windows, end in `.obj`
* **Executable** binary files 
  * generate by `gcc `

### 1.1 Makefile : old school gcc "script"

Make sure the `make` is installed in command-line environment. (added to the system environment variables or `$PATH` in Unix). 

Makefile is like a bash script to automatically compile and solve dependencies for you. (review how a C file is compiled above)

A typical make file look like (all: will link all object files together)

```bash
CC          = g++
CFLAGS      = -std=c++11 -Wall


all: main.o hash.o states.o allStates.o parsetree.o g_array.o hash2.o machine.o transtable.o test.o
	$(CC) $(CFLAGS) -o CSSR main.o hash.o states.o allStates.o parsetree.o g_array.o hash2.o machine.o transtable.o test.o
	

main.o:	Main.cpp Main.h Common.h AllStates.h Machine.h
	$(CC) $(CFLAGS) -c -o main.o Main.cpp

hash.o: Hash.cpp Hash.h States.h Common.h States.cpp
	$(CC) $(CFLAGS) -c -o hash.o Hash.cpp

states.o: States.cpp States.h Common.h ParseTree.h G_Array.h Hash.h
	$(CC) $(CFLAGS) -c -o states.o States.cpp

allStates.o: AllStates.cpp AllStates.h States.h Common.h ParseTree.h G_Array.h
	$(CC) $(CFLAGS) -c -o allStates.o AllStates.cpp

parsetree.o: ParseTree.cpp ParseTree.h Common.h G_Array.h
	$(CC) $(CFLAGS) -c -o parsetree.o ParseTree.cpp

g_array.o: G_Array.cpp G_Array.h Common.h
	$(CC) $(CFLAGS) -c -o g_array.o G_Array.cpp

hash2.o: Hash2.cpp Hash2.h Common.h
	$(CC) $(CFLAGS) -c -o hash2.o Hash2.cpp

machine.o: Machine.cpp Machine.h States.h Common.h Hash2.h Hash.h
	$(CC) $(CFLAGS) -c -o machine.o Machine.cpp

transtable.o: TransTable.cpp TransTable.h Common.h States.h
	$(CC) $(CFLAGS) -c -o transtable.o TransTable.cpp

test.o: Common.h Test.h
	$(CC) $(CFLAGS) -c -o test.o Test.cpp
```

## 2. Matlab Mex

### 2.2 Mex and Makefile 

If `mex` is similar to `g++`, you may have write a makefile as `.m` file. Tips, use the following parameters. 

* `mex -c`  for all C/C++ files 
* `mex -output mexname x.obj y.obj z.obj` to create a MEX file called `mexname` . This is similar to the **Linking** step.



### 2.3 MEX Memory Management

see section **5.1 Matlab Memory Management** of <http://ran.moe/shared/pdf/mex_file_programming.pdf>. 

Memory allocated through `mxMalloc()`  function will be freed automatically when your function ends. 

Hence, if you want to return an output value. Use `mxSetPr`, `mxSetData`, `mxSetString` and `mxCreatString` instead. Otherwise,  the arguments in the left hand side will be always null (Matlab releases the memory space automatically).
