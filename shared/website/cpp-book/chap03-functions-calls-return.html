Functions and Calls

Function Calls

How to pass the parameters as inputs to a called function?

  • Pass-By-Value (default)
  • Pass-By-Pointer
  • Pass-By-Reference (newer C++)
Pass by Value Pass by Pointer Pass by Reference
Copies of the parameter? The value (including memmory address) of each data member is copied when invoking. The address es of the parameters are copied. Nothing
Can make modification to caller's objects? No (except there is pointer in the data member, see shallow-copy-in-pass-by-value) Yes Yes (but it should be avoid by const keyword)
Valid objects passed into the function (compile error instead of runtime error)? Yes NO Yes
Speed? Slow Fast Fast
Safety (side effect) ? Best Worst (pointers are dangerous, use reference if possible). Good

到后面你就大概理解 Pass-by-Value 实际上是 Pass-by-Reference 和 Copy-Constructor.

To even reduce the side-effect the parameter passing, you would better to use const . Read this at Reducing Side-Effect by Const.

Example Custom Class

Assume the common class is Circle.hpp

class Circle{
private:
    double r_;

public:
    Circle(double r); 
    double getArea(); 
    double getR(); 
    void setRadius(double r); 
};

and the implementation is Circle.cpp

#include "Circle.hpp"

Circle::Circle(double r){
  r_ = r;
}

double Circle::getArea(){
  return PI*r_*r_;
}

double Circle::getR(){
  return r_;
}

void Circle::setRadius(double r){
  r_ = r;
}

Pass-by-Value / Pass-by-Object

By default, arguments are "passed by value" to a function. It can also be thought of "Pass-by-Object" as it will invoke the copy constructor.

Assume there is a new function, that will genereate a new circle based on the area 

Circle joinCircle(Circle c1, Circle c2){
    double totalArea = c1.getArea() + c2.getArea();

    double newRadius = std::pow(totalArea/PI, 1.0/2.0);

    Circle result(newRadius);

    /* Some changes to the parameters! */
    c1.setRadius(100);
    c2.setRadius(200);

    return result;
}

Although you make some changes to the data member of the input arguments, but this is pass by value! The arguments here are actually the copies of the original ones. 

int main(int argc, const char * argv[]) {
    Circle c1(10);
    Circle c2(5);

    Circle c3 = joinCircle(c1, c2);

    std::cout << "c3 "<< c3.getR() << std::endl;
    std::cout << "c2 " << c2.getR() << std::endl;
    std::cout << "c1 " << c1.getR() << std::endl;

    return 0;
}

/* Expected: 

    c3 11.1803
    c2 5
    c1 10
*/

Shallow Copy in Pass-by-Value

请记住 Pointer 本身就是一个变量（里面有地址值， 会占用空间）。

But be careful, if the data members are pointers, you only have shallow copy (look at the deep copy and shallow copy)!

/* a class */ 
class Circle{
private:
    double r_;

public:
    Circle(double r){
        r_ = r; 
    }
    int *x;
};

/* to demonstrate the shallow copy since the pointer */ 
Circle modifyPointer(Circle c1){
    /* Some changes to the parameters! */
    *c1.x = 100;
    return c1;
}


int main(int argc, const char * argv[]) {

    Circle c1(10);

    c1.x = new int;
    *c1.x = 0;
    std::cout << "c1.x = " << *c1.x << std::endl;

    Circle c3 = modifyPointer(c1);

    std::cout << "c1.x = " << *c1.x << std::endl;
    return 0;
}

/* Expected: 
  c1.x = 0
  c1.x = 100
*/

Copy Constructor

When a non-primitive variable is passed/returned by value, a copy must be made. As with a constructor, an automatic copy constructor is provided for you if you choose not to define one:

Caution: Copy Constructor

All copy constructors will be triggered by pass-by-values.

All copy constructors will read in the passed parameters by reference.

你可能会被绕晕， 怎么 pass-by-values 里面又有 pass-by-reference 呢？ 你看看 Custom Copy Constructor 就知道怎么回事了。

Automatic copy constructor (shallow copy)

  • Copy each member variable of object using copy constructor
  • Allocate new memory
  • As we dicussed in the last section, the automatic copy constructor is usually shallow copy. see here.

Custom copy constructor (deep copy)

  • read in the source parameters by reference
  • usually defined as deep copy on purpose.

If there is a class Tower, it takes value, pointer and reference

The header file Tower.h looks like 

class Tower{
  public:
      // a custom constructor
      Tower(Cube c, Cube *ptr, const Cube &ref); 
      // a custom copy constructor
      Tower(const Tower &other); 

  private:
      Cube cube_;
      Cube *ptr_;
      const Cube &ref; 
};

And copy constructor (pass-by-reference) looks like 

Tower::Tower(const Tower & other){
  // Deep copy cube_ (copy by value):
  cube_ = other.cube_; 

  // Deep copy ptr_ 
  // the constructor will allocate new space and initialise every thing. and finally return an address
  ptr_ = new Cube(*(other.ptr_));  

  // Deep copy ref_:
  /* NON-SENSE as reference needs an object to refer when it is initialized. 
    you should not have reference in the data member.... 
  */
}

Caution: Never Use Reference in Data Members

The above example may report errors. Because you have reference in the private member. But this is NOT recommended.

#include <iostream>

Reducing Side-Effect by Const

We all like pure things, especially the pure functions as it can reduce our brain memory load!

INFO: Pure and Functional Programming

This is a very important topic in programming language. Look at the functional programming and lambda calculus.

In C++, the side-effect caused by a function can be reduced by using the const keyword.

Pass-by-value-const

Cube joinCubes(const Cube s1, const Cube s2)

Pass-by-pointer-const

Cube joinCubes(const Cube *s1, const Cube *s2)

Pass-by-reference-const

Cube joinCubes(const Cube &s1, const Cube &s2)

Caution: Google C++ Style Guide Best Practice

All parameters passed by reference must be labelled by const.

Const Functions

If you want to promise a function should be pure, you can use const as part of member function's declaration.

The header file Cube.h looks like 

#pragma once
namespace cs225 { 
  class Cube {
  public:
    Cube();
    Cube(double length); 
    double getVolume() const; 
    double getSurfaceArea() const; 
  private:
    double length_; };
}

The implementation Cube.cpp looks like 

double Cube::getVolume() const{
  return length_ * length_ * length_; 
}

double Cube::getSurfaceArea() const{
  return 6 * length_ * length_; 
}

Function Returns

Identical to passing into a function, we also have three choices on how memory is used when returning from a function:

Return by value

Cube joinCubes(const Cube &s1, const Cube &s2)
  • create a new object with constructors.
  • Slow compared to Return-by-reference

Return by pointer

Cube *joinCubes(const Cube &s1, const Cube &s2)
  • Similar to return by value, but the address to the value.

Danger: Stack Frame is Unstable

never return a reference to a local variable within a function (stack memory!)

Return by reference

用法

  • return a reference to *this object
  • want to change the content that is protected a data structure (see in the linked list)
  • faster (without calling constructor)
Cube &joinCubes(const Cube &s1, const Cube &s2)

Danger: Stack Frame is Unstable

never return a reference to a local variable within a function (stack memory!)
